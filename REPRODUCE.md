# Reproducing the results of *Automatic Consistency Assessment through Partial Functional Dependency Mining*

This document describes how to reproduce every figure and table in the paper from the artifacts in this repository.

## 1. Hardware and software used by the authors

All experiments in the paper were run on:

- **CPU:** AMD Ryzen 7 9800X3D (8 physical cores)
- **RAM:** 64 GB DDR5 @ 6000 MHz
- **Storage:** Samsung 980 PRO SSD
- **JVM:** OpenJDK 23 (HyFD / pFD discovery)
- **Python:** 3.10 or newer (Jupyter notebooks for analysis and plotting)

A comparable consumer workstation is sufficient; the most resource-intensive step is pFD discovery on the *Tax* dataset (~2 s in the paper, but memory-bound on larger thresholds).

## 2. Prerequisites

- Git (with submodule support)
- A JDK ≥ 17 (the authors used OpenJDK 23) and the build tool documented in the HyFD submodule
- Python ≥ 3.10 with Jupyter, for the analysis notebooks

## 3. Clone the repository (with submodules)

The pFD discovery code and the pollution generator both live in separate repositories and are included as Git submodules:

- `HyFD/` — fork of HyFD on the `gpdep` branch (pFD discovery + `gpdep` weighting)
- `experiments/Dataset-Pollution-Generator/` — `tab_err`-based pollution pipeline used in Sec. 4.2

```bash
git clone --recurse-submodules https://github.com/SeegerM/cpfd-reproducibility.git
cd cpfd-reproducibility
```

If you already cloned without `--recurse-submodules`:

```bash
git submodule update --init --recursive
```

## 4. Build the pFD discovery tool (HyFD, `gpdep` branch)

The submodule is a fork of [HyFD](https://github.com/SeegerM/HyFD) on the `gpdep` branch, extended with:

- threshold-and genuineness-based validation for partial FDs (Sec. 3.1 of the paper), and
- on-the-fly `gpdep` genuineness computation (Sec. 3.2).

Build instructions, CLI flags, and output format are documented in the submodule's own README:

> See [`HyFD/README.md`](HyFD/README.md).

The `data/<dataset>/clean/<dataset>.txt` files shipped here were produced from `clean.csv` with the threshold `ρ_max = 0.95` used throughout the paper. Each output line has the form

```
<lhs> -> <rhs> [measure=<ρ>, gpdep=<ω̃>]
```

where `measure` is the partial validity `ρ` and `gpdep` is the un-normalized genuineness score; the normalized weight `ω` defined in Sec. 3.2 of the paper is applied inside the analysis notebooks.

## 5. Set up the Python environment

```bash
python -m venv .venv
# Windows
.venv\Scripts\activate
# macOS / Linux
source .venv/bin/activate

pip install pandas numpy scipy matplotlib seaborn jupyter
jupyter lab            # or: jupyter notebook
```

The pollution generator submodule has its own dependencies; see [`experiments/Dataset-Pollution-Generator/README.md`](experiments/Dataset-Pollution-Generator/README.md).

## 6. Datasets

The seven evaluation datasets are bundled in `data/`. For each dataset, `clean/clean.csv` is the cleaned input and `clean/<dataset>.txt` contains all pFDs with `ρ ≥ 0.95` and their `gpdep` weights as produced by the HyFD fork.

For Section 4.2, polluted variants are generated by the [`Dataset-Pollution-Generator`](experiments/Dataset-Pollution-Generator/) submodule, which wraps the [`tab_err`](https://pypi.org/project/tab-err/) library (Jung et al., 2025). The submodule ships:

- per-dataset error **profiles** in `profiles/` (registered in `registry.py`),
- the error mechanisms (ECAR / EAR / ENAR) and error types used in the paper, and
- the random seed used by the authors (`seed=7` for the hospital pipeline in `main.py`).

Refer to the submodule README for invocation; in short:

```bash
cd experiments/Dataset-Pollution-Generator
python main.py    # runs the default Hospital pipeline used in Sec. 4.2
```

## 7. Reproducing each result

| Paper artifact                       | Section | Notebook / tool                                       |
| ------------------------------------ | ------- | ----------------------------------------------------- |
| Table 2 — discovery runtimes         | 4.1     | `experiments/Runtime Comparison.ipynb`                |
| Figure 2 — attribute coverage        | 4.1     | `experiments/Coverage.ipynb`                          |
| Polluted-dataset variants            | 4.2     | `experiments/Dataset-Pollution-Generator/` (submodule)|
| Figure 3 — metric vs. pollution stage| 4.2     | `experiments/Metric Polution.ipynb`                   |
| Table 3 — Pearson correlations       | 4.2     | `experiments/Correlation.ipynb`                       |

All notebooks write their generated CSVs and plots into `experiments/results/`. Reference outputs (`corr_matrix_p.{csv,pdf,png}`, `corr_matrix_r.{csv,pdf,png}`, `variant_level_metrics.csv`) are included for comparison.

### Suggested run order

1. `Runtime Comparison.ipynb` — sanity-checks that the HyFD build works and produces the per-dataset `.txt` files.
2. `Coverage.ipynb` — Figure 2.
3. `Dataset-Pollution-Generator` — regenerate the Hospital pollution variants.
4. `Metric Polution.ipynb` — Figure 3.
5. `Correlation.ipynb` — Table 3.
